Initial Setup

Getting Your OpenRouter API Key

One API, Multiple Models: OpenRouter gives you access to all major AI models through a single API key, making comparisons seamless and cost-effective.

  1. 1.
    Visit openrouter.ai and create a free account
  2. 2.
    Go to your dashboard and navigate to "API Keys"
  3. 3.
    Click "Create Key" and give it a name like "LLM Comparator"
  4. 4.
    Copy your API key (starts with "sk-or-") and keep it secure

Configuring the Comparator

  1. 1.
    Open the LLM Response Comparator
  2. 2.
    Click the ⚙️ settings button in the top-right corner
  3. 3.
    Paste your OpenRouter API key in the designated field
  4. 4.
    Select 2-4 models you want to compare (start with 2 for simplicity)
  5. 5.
    Click "Save" to store your configuration

Privacy Note: Your API key is stored locally in your browser and only used to communicate directly with OpenRouter. No data is sent to any other servers.

Running Comparisons

Step-by-Step Process

  1. 1.
    Enter Your Prompt: Type or paste your prompt in the large text area. Be specific about what you want to achieve.
  2. 2.
    Review Selected Models: Ensure you have 2-4 models selected for comparison.
  3. 3.
    Click "Run Comparison": The tool will send your prompt to all selected models simultaneously.

Analyzing Results

Learn how to interpret and compare the responses to make informed decisions about model selection.

Quality Assessment

Accuracy

Check factual correctness and relevance to your prompt

Clarity

Evaluate how well-structured and understandable the response is

Completeness

Does the response fully address all parts of your prompt?

Style

Consider tone, formality, and appropriateness for your use case

Performance Metrics

Response Time

Faster models (usually 1-3 seconds) are better for interactive applications. Slower models (3-10 seconds) often provide more thoughtful responses.

Token Usage

Higher token counts mean longer responses and higher costs. Consider whether the extra length provides proportional value.

Common Comparison Patterns

Claude vs GPT-4o

Claude often provides more structured, analytical responses. GPT-4o tends to be more creative and conversational.

Expensive vs Budget Models

Premium models (Opus, GPT-4o) excel at complex reasoning. Budget models (Haiku, Mini) are great for simple tasks.

Specialized vs General Models

Some models excel in specific domains (coding, writing, analysis). Test with domain-specific prompts.

Making Your Decision

Decision Framework:

  1. 1. Quality First: Which response best meets your requirements?
  2. 2. Consistency: Test the same prompt multiple times - which model is most reliable?
  3. 3. Speed Requirements: Do you need real-time responses or can you wait for quality?
  4. 4. Cost Considerations: Factor in token usage and pricing for your volume
  5. 5. Use Case Fit: Which model aligns best with your specific application?

Remember: The "best" model depends entirely on your specific use case. A model that excels at creative writing might not be ideal for technical analysis.

Troubleshooting

API Key Issues

Error: "Please enter your OpenRouter API key and save settings"

Solutions:

  • • Verify your API key starts with "sk-or-" and is entered correctly
  • • Check that you clicked "Save" after entering the key
  • • Ensure your OpenRouter account has sufficient credits
  • • Try refreshing the page and re-entering your settings

Comparison Failures

Some models show "Error" messages instead of responses

Solutions:

  • • Check your internet connection stability
  • • Verify you have access to the specific models (some require higher account tiers)
  • • Try with a simpler prompt to test basic functionality
  • • Reduce the number of models being compared simultaneously
  • • Check OpenRouter status page for service issues

Performance Issues

Slow responses or timeouts during comparisons

Solutions:

  • • Compare fewer models at once (start with 2 instead of 4)
  • • Use shorter prompts for testing
  • • Check if specific models are consistently slow
  • • Try during off-peak hours when API load is lower
  • • Clear browser cache and restart the application

History/Storage Problems

Comparisons not saving or history not loading

Solutions:

  • • Check if local storage is enabled in your browser
  • • Exit private/incognito mode (storage is limited there)
  • • Clear browser data and start fresh if storage is corrupted
  • • Export important comparisons before clearing storage
  • • Try a different browser to isolate the issue

Model Selection Issues

Can't select certain models or getting "model not available" errors

Solutions:

  • • Check your OpenRouter account tier - some models require paid plans
  • • Verify the model IDs are correct and still supported
  • • Try selecting different model combinations
  • • Check OpenRouter documentation for model availability
  • • Contact OpenRouter support for account-specific issues

Still Need Help?

Check Browser Console: Press F12 and look in the Console tab for detailed error messages.

Test with Simple Prompts: Try "Hello, how are you?" to verify basic functionality.

OpenRouter Resources: Visit the OpenRouter documentation and status page for service updates.

Browser Compatibility: Ensure you're using a modern browser with JavaScript enabled.
